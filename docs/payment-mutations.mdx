# Payment Mutations

Mengambil daftar mutasi atau transaksi masuk dari rekening Anda yang berhasil terdeteksi oleh Android Worker.

Data mutasi ini sangat berguna untuk keperluan rekonsiliasi serta pencocokan transaksi secara manual apabila proses otomatisasi mengalami kendala.

**Cara kerja:** Android Worker mendeteksi notifikasi transfer masuk → mengirimkan data ke server → sistem secara otomatis mencocokkan transaksi tersebut dengan *invoice* berstatus `PENDING` berdasarkan nominal unik. Jika nominal cocok, status *invoice* akan berubah menjadi `PAID`. Jika tidak ada nominal yang sesuai, mutasi tersebut akan berstatus `unmatched` dan dapat dicocokkan secara manual melalui *endpoint* [Payment Match](./payment-match).

Gunakan parameter `only_unmatched=true` untuk memfilter dan hanya menampilkan mutasi yang belum terpasangkan secara otomatis.
